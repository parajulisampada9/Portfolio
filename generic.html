Project Overview

Planning and Implementation

AWS Honeypot

This project will leverage AWS to demonstrate how to effectively secure cloud environments using industry-standard tools and procedures. We will need an AWS account and an EC2 instance to carry out the operation. Services such as Virtual Private Cloud (VPC) will be used to create a virtual network, and Security Groups will be used to route traffic to expose the server to the internet. Outbound traffic restrictions will be applied to prevent external interference and further attacks. Once we begin gathering data, we will use a Python script to push the logs to an S3 bucket, which will trigger AWS Lambda, which will parse the logs to extract metadata. From there, we will write the parsed data to AWS DynamoDB and a tool like Streamlit to provide a visual dashboard that displays some of the key information that we have gathered. Tools like CloudTrail, Cloudbridge, and AWS Health will also be considered for effective logging and alert mechanisms depending on the scope of the project. The cloud environment will operate for approximately a week. The defense mechanism of the honeypot system will be deliberately weak to lure in attackers. Decoy files will be used instead of legit sensitive files as an incentive for the attackers to infiltrate the system. These files will not contain malware or anything similar. They are not designed to harm the attackers, just to make the honeypot environment more believable and realistic. The environment will operate in a VPC (Virtual Private Cloud) and any attempt to get access to the private network will be recorded. During the set time period, the logs will be consistently monitored to detect anything out of normal. Once an attack is discovered, logs will be analyzed to identify the attacker’s metadata.
